# コミットしてプッシュ

変更をコミットして GitHub にプッシュしてください。

## 手順

1. `git status` で変更内容を確認

2. 変更内容に基づいて適切な gitmoji を選択:
   - `🎉` 初期コミット
   - `✨` 新機能を追加
   - `🐛` バグを修正
   - `📝` ドキュメントを更新
   - `♻️` コードをリファクタリング
   - `🔧` 設定ファイルを変更
   - `🧪` 実験を追加
   - `📊` 実験結果を記録
   - `🎨` コードのスタイル・フォーマットを改善
   - `🚀` デプロイ関連
   - `➕` 依存関係を追加
   - `🔥` コード・ファイルを削除

3. 日本語でコミットメッセージを作成
   - 形式: `{gitmoji} {変更内容の要約}`
   - 例: `🧪 exp001-baseline を追加`
   - 例: `📊 exp001 の結果を記録（CV: 0.85, LB: 0.83）`

4. 変更をステージング（`git add`）
   - 関連するファイルのみをステージング
   - 不要なファイルが含まれていないか確認

5. コミットを作成

6. GitHub にプッシュ

## 引数

$ARGUMENTS にコミットメッセージが渡される場合は、それを使用してください。
ただし gitmoji がない場合は適切なものを先頭に追加してください。

## 完了後

- コミットハッシュとメッセージを報告
- プッシュ先の URL を報告
